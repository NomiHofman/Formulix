# FORMULIX - דוח מסכם
## מערכת השוואת מנועי חישוב דינמיים

**תאריך:** אפריל 2026  
**מחבר:** פרויקט מבדק פיתוח רמה ג' - משרד החינוך

---

## תקציר מנהלים

פרויקט FORMULIX פותח במטרה להשוות בין שיטות שונות לחישוב נוסחאות דינמיות על מיליון רשומות נתונים. המערכת מדגימה כיצד ניתן לקבוע תעריפים ונוסחאות באופן דינמי, ללא שינוי בקוד המקור, תוך שמירה על ביצועים גבוהים.

**ממצא מרכזי:** שיטת **SQL Dynamic** מספקת את הביצועים הטובים ביותר עם זמן ריצה ממוצע של **9.8 שניות** על מיליון רשומות, בעוד כל השיטות מייצרות תוצאות **זהות לחלוטין**.

---

## 1. רקע ומטרות

### 1.1 האתגר העסקי
מערכות תשלומים מודרניות נדרשות לחשב מאות סוגי תעריפים שונים. התעריפים והנוסחאות משתנים באופן תדיר בהתאם ל:
- חוקים ותקנות חדשים
- הסכמים עם לקוחות
- עונתיות ותאריכים
- סוגי לקוחות שונים

### 1.2 הפתרון
במקום לקבע כל נוסחה בקוד, המערכת לומדת נוסחאות חדשות **בזמן ריצה** ישירות מטבלת נוסחאות במסד הנתונים. גישה זו מאפשרת:
- שינוי נוסחאות ללא עדכון קוד
- התאמה מהירה לשינויים
- גמישות מרבית בהגדרת חישובים

## 2. ארכיטקטורת המערכת

### 2.1 מבנה הנתונים

```
┌─────────────────┐    ┌─────────────────┐    ┌─────────────────┐
│     t_data      │    │    t_targil     │    │   t_results     │
├─────────────────┤    ├─────────────────┤    ├─────────────────┤
│ data_id (PK)    │    │ targil_id (PK)  │    │ results_id (PK) │
│ a (FLOAT)       │    │ targil (נוסחה)   │    │ data_id (FK)    │
│ b (FLOAT)       │    │ tnai (תנאי)      │    │ targil_id (FK)  │
│ c (FLOAT)       │    │ targil_false    │    │ method          │
│ d (FLOAT)       │    └─────────────────┘    │ result          │
└─────────────────┘                           └─────────────────┘
     1,000,000                 20                   
     רשומות                  נוסחאות                
```

### 2.2 סוגי נוסחאות נתמכים

| סוג | דוגמה | תיאור |
|-----|-------|-------|
| פשוטה | `a + b` | פעולות אריתמטיות בסיסיות |
| מורכבת | `SQRT((c*c) + (d*d))` | פונקציות מתמטיות (sqrt, log, abs, pow) |
| תנאי | `if(a > 5, b * 2, b / 2)` | חישוב מותנה בהתאם לערכים |

## 3. שיטות החישוב

### 3.1 SQL Dynamic (sp_executesql)

**תיאור:** מעביר את חישוב הנוסחה ישירות למנוע מסד הנתונים.

**יתרונות:**
- ✅ הביצועים הטובים ביותר (~10 שניות)
- ✅ אין העברת נתונים ברשת
- ✅ אופטימיזציות מובנות של SQL Server

**חסרונות:**
- ❌ תלות בסינטקס SQL Server
- ❌ קשה יותר לדבג נוסחאות מורכבות

### 3.2 Roslyn C# Scripting

**תיאור:** מקמפל את הנוסחה לקוד C# נייטיבי ומריץ על כל רשומה.

**יתרונות:**
- ✅ גמישות מלאה - כל קוד C# אפשרי
- ✅ קל לדבג ולבדוק
- ✅ אין תלות במסד נתונים ספציפי

**חסרונות:**
- ❌ איטי יותר (~20 שניות)
- ❌ צורך יותר זיכרון

### 3.3 Python SymPy + NumPy

**תיאור:** משתמש בספריית SymPy לפרסור וב-NumPy לחישוב וקטורי.

**יתרונות:**
- ✅ חישוב וקטורי מהיר
- ✅ תמיכה בכל פונקציה מתמטית
- ✅ קוד קריא ופשוט

**חסרונות:**
- ❌ דורש התקנת Python
- ❌ העברת נתונים בין טכנולוגיות

### 3.4 AI Translated (OpenAI GPT)

**תיאור:** שולח את הנוסחה ל-GPT לתרגום אוטומטי ל-C#.

**יתרונות:**
- ✅ מבין נוסחאות בשפה טבעית
- ✅ גמיש לפורמטים שונים
- ✅ מדגים שימוש חדשני ב-AI

**חסרונות:**
- ❌ תלות בשירות חיצוני
- ❌ עלות לכל קריאת API
- ❌ לא דטרמיניסטי

| AI Translated | 22.1s | 19.8s | 26.3s |

```
✓ כל השיטות מייצרות תוצאות זהות
✓ סבילות: 1e-9 (הבדלי נקודה צפה)
✓ נבדקו 12,000,000 תוצאות (12 נוסחאות × 1M רשומות × 4 שיטות)
```

## 8. סיכום

פרויקט FORMULIX הדגים בהצלחה את היתרונות של מערכת חישוב נוסחאות דינמית:

1. **גמישות** - נוסחאות ניתנות לשינוי ללא עדכון קוד
2. **ביצועים** - חישוב מיליון רשומות בפחות מ-10 שניות
3. **דיוק** - כל השיטות מייצרות תוצאות זהות
4. **ניידות** - תמיכה במספר טכנולוגיות וגישות

המערכת מוכנה לשימוש בסביבת ייצור עם ההמלצה להשתמש בשיטת SQL Dynamic כברירת מחדל.
